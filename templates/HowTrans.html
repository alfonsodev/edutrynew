How to Translate Khan Academy Platform Content

Please note that the screen shots for this documentation were captured at different stages of the development process and may therefore not always match what you might see on your screen in every detail. Where the change is important to the point being discussed in that part of the documentation we have change the image, but some may have been missed. Please submit feedback if you have any suggestions for improving this documentation.

If you have not yet done so, please read the paper Basic Architecture of the Khan Academy Exercise Platform under the Menu Item Learning Platform / Discussion Papers.



What Needs to be Translated:

Translating an exercise therefore generally consists of translating the following:

  1. The visible text in the Site Template (file: khan-site.html) - done once for all exercises.
  2. The visible text in the Exercise Template (file: khan-exercise.html) - done once for all exercises.
  3. The visible text in the individual exercise files: done once for each applicable exercise (some exercises have no text, and some, e.g. word problem exercises have text that is too complicated for simple translation).
  4. The visible text in some of the utility functions (in .js files) (In some cases, exercise files call utility functions and in some cases, those functions return text content.)
  5. The visible text in the Report Templates (the container templates for the various coach and learner reports).
  6. Etc.



Overview of The Translation Process:



Create Template Records:

This is an admin function that is not available to most users.



Tokenize Files and Create Token Records in Database:

This is an admin function that is not available to most users.

The process begins with the admin step of tokenizing the original html, js and other files which we generally just refer to as template files. This is done as an admin step. (Tokens are a key concept to this translation process. Tokens are placeholders in the template files that are replaced by the content in the applicable language from the database when the translated file is generated. Tokenizing is the process replacing the original English content with the tokens - e.g., tkn001.) In some cases, this is an automated step where the files are automatically parsed and the tokenization and the entry of the tokens into the database are automated. It is envisioned that there would eventually be a component that would recognize any time a file in the English version was added or changed and these would be entered into a queue for adding to this system. This is only until Khan Academy comes up with a better way to manage this process (hopefully).



Starting the Translation Process:

This is where the process starts for most users involved in the translation process.

The Gateway to the Translation process is via the Learning Platform menu item.



Status of the overall Translation Effort:

The first screen after clicking on the Exercise menu item presents a summary of the number of tokens for each Template File (English view). (There is also a link to adding a new token. This is an admin function that is not available to most users.)



Viewing the Tokens for a Template:

Click on the name of the Template to see the Tokens for that template. (There is also a link on this page to add new tokens. If you are in the English Version, then this is an admin function that is not visible to most users - the English language version may only be administrated by users with special rights.)



Status of Translation for a particular Language:

On the Template List page, change the language to the desired language.

This will result in an overview of the status for all the templates for that language. Including the number of tokens that have been cloned into that language (cloned just means there is a database record for that token - it does not indicate which tokens have been translated yet and which not). You will see the status of the tokens for a template when you drill into the template.

If the number of tokens in the selected language is less than the number of English language tokens, this means that some tokens may have been added to the English language version since the last time the tokens were cloned to that language for that template. If this is the case, click on the "clone" link to clone the missing tokens for this template. (Only tokens not yet in that language will be cloned - tokens that were previsously cloned are not affected.) The clone function will only be visible if the number of tokens in the selected language is less than the number of English tokens for that template.



Status of Tokens for a Template for a specific Language:

Click on the name of the Template to see the Tokens for that template as well as the status of each token. Tokens may be "Pending Translation", Pending review" or "Published". Click on the Edit link for a token to view or edit/translate the details for that template. (Note that the Generate Translated Files link is no longer visible until all tokens are published and the View Translated Files is not visible unless there is a translated file for this template.



Translating the Tokens for a Template:

Click on the edit link to drill into an individual token to translate it. You will see the original English content as well as the current content for the selected language. Translate/edit the content as applicable. Change the status as needed. (Currently a user can change the status to any of the statuses. We are thinking about making it that one can change the status only 1 level at a time - so the user doing the translation would only be able to change the status to Pending review but not to published. This could be part or requiring the review be done by someone other than the person who did the translation. We are not making this change now because many languages do not have the number of people needed to require different people do different functions.)

Generate Translated Template File:

When all the tokens fro a template have a status = "Published" a "Generate Translated File" Link will appear. Click on this link to generate the translated file. (Note: If a translated file already exists, it will be overwritten by this process.) (The translated file is actually a blob field in the database. At some point it will be exported to generate the translated site.)

If a Translated File exists for this template, you will see a View Translated File link. Click this link to see what the file will look like after translation.

Try Out Translated Exercises:

We hope to be able to incorportate an online exercise framework that will allow users to actually try out the translated exercise before exporting it. This means a user will be able to translate a single exercise and actually try it out before translating any other exercises. This functionality is not yet available but we hope it can be made available soon.



Export Translated Site:

This functionality is not yet available. We wanted to get the main site out as quickly as possible so that we can ensure we are moving in the right direction before working on some of the final steps.
