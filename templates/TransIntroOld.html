Welcome to the Translation Section. NOTE: The Structure of the Translation Efforts has been sigificantly changed and this Intro will be rewritten shortly.

A major part of this site is dedicated to supporting the efforts to make the Khan Academy Platform available in any language. Almost everything related to the platform translation effort can be found under the Platform Menu Item (where else):

  1. Topic/Learning Unit Names: The names that are visible on the Knowledge/Learning Map.
  2. Templates: This is the name of the files that have language-related content. The language related content has been replaced with placeholders which we are calling tokens. There are database records for each token for each language. Once all the tokens for a particular language have been translated in the database, a translated file can be generated whereby the tokens are replaced with the translated text for the applicable language. See How to Translate Exercise Platform Content for detailed documentation of the steps to translating exercises. Templates fall into the following categories:
    1. Page Content: The templates that contain the static content on pages.
    2. Container Templates: These are the parts of a web page surrounding other templates (currently the site-template and the exercise-wrapper template).
    3. Exercise Templates: These are the templates that contain the actual content for individual exercises.
    4. Function Templates: These are the javascript files that generate content that go into other templates, in particular the Exercise Templates.
    5. Report Templates: These are the templates that contain report content for learners and coaches.
    6. Etc.

The basic concept behind the translation of the Exercise Platform is that every language will have its own set of files that mirror the original Khan Academy files, except in their own language. The purpose of the application functionality of this site is to make the process of translating and managing the translations as easy as possible and minimizing the need for a technical background.
